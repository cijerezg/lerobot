# $\pi_{0.5}$ specific notes

This page collects $\pi_{0.5}$-specific setup that is not needed for MolmoAct2 (the default policy in this repo). The training/inference scripts are the same — only the dataset prep and the `policy.type` field differ.

## Dataset: subtask annotations

$\pi_{0.5}$ expects per-frame subtask labels in the dataset. At least 50 episodes annotated with subtasks are recommended.

The subtask annotation can be done manually:

```bash
uv run python -m lerobot.policies.pi05_full.annotate.manual_subtask_annotate
```

This launches a web interface to annotate the subtasks. Another tool for manual subtask and general dataset editing is available at [lerobot-data-studio](https://github.com/jackvial/lerobot-data-studio).

The other option is LLM annotation. We recommend Gemma 4:

```bash
uv run python -m lerobot.policies.pi05_full.annotate.subtask_annotate_gemma_4 \
    --data-dir /path/to/your/dataset \
    --video-key observation.images.wrist \
    --batch-size 5 \
    --output-dir /output/path
```

## Config

Set `policy.type: pi05_rl` and point `base_path` at the base $\pi_{0.5}$ weights (e.g. `lerobot/pi05_base` or a local copy). All other fields work the same as the MolmoAct2 flow described in the main README.
